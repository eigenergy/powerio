# pandapower fixture

`example.json` was generated by pandapower 3.2.2 (`pp.to_json`) and is committed
byte exact as the tool wrote it. It is `pandapower.networks.example_simple()`
(7 buses with a 0-based index, ext_grid, gen, sgen, trafo, shunt, load, 4 lines,
8 switches) plus a storage unit and a dcline:

```python
import pandapower as pp, pandapower.networks as pn
net = pn.example_simple()
pp.create_storage(net, bus=6, p_mw=0.5, max_e_mwh=2.0, q_mvar=0.1)
pp.create_dcline(net, from_bus=3, to_bus=4, p_mw=2.0, loss_percent=1.0,
                 loss_mw=0.05, vm_from_pu=1.01, vm_to_pu=1.0)
pp.to_json(net, "tests/data/pandapower/example.json")
```

Regenerate with the same pandapower version (the CI pin in
`benchmarks/requirements.txt`); a different version reshuffles column order and
defaults, which breaks byte level comparisons.
