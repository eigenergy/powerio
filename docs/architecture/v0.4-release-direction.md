# v0.4.0 release direction: multiconductor to balanced lowering

This note is the design contract for the v0.4.0 direction around lowering a
`MulticonductorNetwork` into a `BalancedNetwork`. The implementation is tracked
in #145.

## IR boundary

`BalancedNetwork` and `MulticonductorNetwork` stay distinct concrete IR families.
Under the v1 naming direction in #144, the top level `Network` name is the
static grid IR container. It carries one concrete payload at a time; it does not
flatten balanced and multiconductor fields into one electrical model. The
lowering is an explicit pass from the wire coordinate distribution model to the
scalar positive sequence model. It is not a merger of the two model structs, and
it is not an implicit conversion hidden inside package readback, format
conversion, matrix builders, or bindings.

The pass must produce a new balanced package payload and append a
`LoweringRecord` to `lowering_history`. The record is the public audit trail for
the pass: options, assumptions, approximations, dropped fields, diagnostics, and
final validation status.

## Supported starting point

The first supported input class is transparent three phase input where the
sequence projection is defined without guessing. A network with phase conductors
plus a neutral conductor is handled by applying Kron reduction before the
sequence transform. One wire and two wire inputs are not silently projected into
positive sequence; they must return structured diagnostics that reject the
lowering or explain the approximation if a later policy permits one.

The implementation must choose and document the sequence transform convention
before it emits balanced data. That convention must state whether the transform
is power invariant. Tests and diagnostics should refer to the convention by
name, so downstream users can tell which projection produced the result.

## Diagnostics and provenance

The pass must surface modeling decisions as structured findings instead of
quietly rewriting the model. Diagnostics should cover unsupported conductor
sets, ambiguous one wire or two wire inputs, lossy phase aggregation, dropped
grounding or conductor detail, and any source object that cannot be represented
in `BalancedNetwork`.

The lowered package should also preserve provenance:

- `lowering_history` records the pass name, input and output model kinds,
  transform convention, assumptions, approximations, dropped fields, diagnostics,
  and validation status.
- `source_maps` use `mapping_kind = lowered`, `aggregated`, `synthetic`, or
  `retained_extra` where balanced fields do not map exactly to one source field.
- `diagnostics` use `LOWER.*` codes for pass findings and `VALIDATE.*` codes for
  validation findings after lowering.

## Follow up

#145 tracks the lowering API, supported case set, and tests. That work should
start from the contract above and keep unsupported cases explicit until a public
policy says otherwise.
